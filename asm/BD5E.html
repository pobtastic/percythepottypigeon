BD5E: Draw Sprite Column
Used by the routine at Draw_Sprites.
Input
A Shift amount (self-modified into the code)
HL Screen buffer address
IX Pointer to sprite data (IX+03 = one-indexed frame number)
Self-modify the shift loop counter at BD82.
DrawSpriteColumn BD5E LD ($BD82),A Write A to *BD82.
Look up the sprite frame graphic data. Each frame is 20 bytes, so multiply the zero-indexed frame number by 20 and add the sprite sheet base address.
BD61 PUSH HL Stash the screen buffer address on the stack.
BD62 LD DE,$AB3B DE=Sprite_01 (sprite sheet base address).
BD65 LD L,(IX+$03) L=*IX+03 (one-indexed frame number).
BD68 DEC L Decrement to make it zero-indexed.
BD69 LD H,$00 Create an offset in HL using the frame index.
BD6B ADD HL,HL
BD6C ADD HL,HL Multiply HL by 20.
BD6D ADD HL,HL
BD6E ADD HL,HL
BD6F ADD HL,HL
BD70 ADD HL,DE Add the sprite sheet base address.
BD71 EX DE,HL Exchange so DE points to the sprite graphic data.
BD72 POP HL Restore the screen buffer address from the stack.
Draw 02 columns, each 10 pixel rows tall. C holds a mask of 07 used to detect character cell boundaries for screen address adjustment.
BD73 LD B,$02 Set a column counter in B of 02 columns.
DrawSpriteColumn_ColumnLoop BD75 LD C,$07 C=07 (pixel row mask for character cell boundary detection).
DrawSpriteColumn_0 BD77 PUSH BC Stash the column counter and screen buffer address on the stack.
BD78 PUSH HL
BD79 LD B,$10 Set a row counter in B of 10 pixel rows.
For each row, read a byte of sprite data, shift it left by the self-modified amount and OR the resulting two bytes across two adjacent screen buffer columns.
DrawSpriteColumn_RowLoop BD7B PUSH DE Stash the sprite data pointer and screen buffer address on the stack.
BD7C PUSH HL
Read the sprite data byte and shift it into a 16-bit value in HL.
The shift amount at BD82 was self-modified on entry to control the horizontal pixel alignment.
BD7D LD A,(DE) A=*DE.
BD7E LD L,A Transfer sprite byte to HL (clearing the high byte).
BD7F LD H,$00
BD81 JR DrawSpriteColumn_ShiftLoop Jump into the shift loop at DrawSpriteColumn_ShiftLoop.
DrawSpriteColumn_ShiftExtra BD83 ADD HL,HL Shift HL left one position.
DrawSpriteColumn_ShiftLoop BD84 ADD HL,HL Shift HL left a further seven positions (eight total from DrawSpriteColumn_ShiftExtra, or seven from DrawSpriteColumn_ShiftLoop).
BD85 ADD HL,HL
BD86 ADD HL,HL
BD87 ADD HL,HL
BD88 ADD HL,HL
BD89 ADD HL,HL
BD8A ADD HL,HL Stash placeholder (last shift).
OR the shifted sprite data onto the screen buffer. The high byte goes
into the current column and the low byte into the adjacent column.
BD8B POP DE Restore the screen buffer address into DE, keeping a copy on the stack.
BD8C PUSH DE
BD8D EX DE,HL Exchange so HL=screen buffer address, DE=shifted sprite data.
BD8E LD A,D OR the high byte of the sprite data onto *HL.
BD8F OR (HL)
BD90 LD (HL),A
BD91 INC L Move to the next column and OR the low byte of the sprite data onto *HL.
BD92 LD A,E
BD93 OR (HL)
BD94 LD (HL),A
BD95 POP HL Restore the screen buffer address and sprite data pointer from the stack.
BD96 POP DE
Move down one pixel row in the screen buffer and check for a character cell boundary crossing.
BD97 INC H Move down one pixel row in the screen buffer.
BD98 LD A,H Test if we've crossed a character cell boundary (every 08 rows).
BD99 AND C
BD9A CALL Z,$BC37 Call BC37 to adjust the screen buffer address if a character cell boundary was crossed.
Move to the next sprite data byte and loop.
BD9D INC DE Advance the sprite data pointer.
BD9E DJNZ DrawSpriteColumn_RowLoop Decrease the row counter by one and loop back to DrawSpriteColumn_RowLoop until all 10 rows are drawn.
Move to the next column in the screen buffer and loop.
BDA0 POP HL Restore the screen buffer address from the stack.
BDA1 POP BC Restore the column counter from the stack.
BDA2 INC L Move one byte to the right in the screen buffer.
BDA3 DJNZ DrawSpriteColumn_0 Decrease the column counter by one and loop back to DrawSpriteColumn_ColumnLoop until both columns are drawn.
BDA5 RET Return.
